# PRD — 오장 내기 정산 계산기

## 1. 개요

골프 라운드 중 카트에서 폰으로 사용하는 **오장(타당 정산) 내기 자동 정산 웹앱**.

내기 룰은 조마다, 라운드마다 달라진다. 따라서 **라운드 시작 전에 룰 설정 체크리스트**를 한 번 입력하면, 이후에는 **매홀 각자 타수(+니어/롱기 수령자)만 입력**하면 배판 판정·버디값·니어 보상을 자동 반영해 "지금까지 누가 누구에게 얼마"를 실시간으로 보여준다.

### 사용자 시나리오

1. 총무가 티오프 전 앱을 열고 플레이어 이름과 룰(아래 체크리스트)을 설정한다.
2. 매홀 끝날 때 각자 타수를 입력한다. 파3이면 니어 수령자, 파5면 롱기 수령자를 선택한다.
3. 앱이 배판 여부를 자동 판정해 다음 홀에 뱃지로 표시하고, 누적 정산표를 갱신한다.
4. 몸풀기 홀 등은 "이 홀 내기 제외" 토글로 스코어만 기록하고 정산에서 뺀다.
5. 라운드 종료 후 최종 정산표(최소 송금 요약 포함)를 확인하고, 카톡 공유용 텍스트를 복사한다.

### 제약/전제

- **정산 방식은 홀별 쌍별(pairwise) 타수차 정산** (§4 참조).
- 총무 **1명이 입력**한다 (멀티 디바이스 동기화 없음).
- **서버 없음** — 모든 데이터는 브라우저 localStorage에 저장. 앱이 꺼져도 진행 중 라운드 복구.
- 모바일 우선 UI (카트에서 장갑 낀 손으로도 누를 수 있는 큰 터치 타깃).
- 배포는 추후 결정(GitHub Pages/Vercel 등 무료 정적 호스팅). PWA로 홈 화면 설치 + 오프라인 동작 지원.

## 2. 시작 전 룰 설정 체크리스트 (설정 화면 스펙)

모든 항목은 기본값을 가지며, 설정 화면에서 라운드 시작 전에 확인/수정한다.

### 2.1 기본

| 항목 | 기본값 | 비고 |
|---|---|---|
| 인원 | 4명 | 3~4명 지원 |
| 플레이어 이름 | P1~P4 | 직접 입력 |
| 타당 금액 | 5,000원 | 1타 차이당 주고받는 금액 |
| 버디값 | 5,000원 | 버디한 사람이 **나머지 각자에게서** 받는 금액 |
| 이글값 | 10,000원 | 〃 |
| 알바트로스/홀인원값 | 0원(끔) | 〃. 보통 별도 한턱이라 기본 끔 |

> 오장에서 **양파(더블파) 이상은 없다** — 타수 입력 자체가 양파까지로 제한되며 항상 양파로 정산 (옵션 아님).
> 첫홀 몸풀기는 별도 옵션 없이 홀별 "내기 제외" 토글(§2.6)로 처리.

### 2.2 핸디

**플레이어별 "받는 타수" 하나만 입력** — 쌍별 차이는 자동 적용된다.

- 예) 나 0타, B 0타, C 5타, D 10타 입력 시: 나→C 5타, 나→D 10타, B→C 5타, B→D 10타, C→D 5타가 자동 성립
- 금액 환산: 차이 타수 × 타당 금액 (10타 × 5,000원 = 50,000원)
- **라운드 시작부터 누적 손익에 즉시 반영** — 홀을 치기 전에도 D +50,000 / 나 −50,000처럼 표시
- 정산표에 "핸디" 항목으로 별도 표기

> v2 후보: 매홀 분산 차감(특정 홀에 핸디 타수 배정) 방식.

### 2.3 배판

| 항목 | 기본값 | 비고 |
|---|---|---|
| 버디 시 다음홀 배판 | 켬 | 버디 이상(이글 포함) 나오면 다음 홀 배판 |
| 양파·양파직전 시 다음홀 배판 | 켬 | 파3 더블보기(5), 파4 트리플(7), 파5 쿼드(9) 이상 |
| (인원−1)명 동타 시 당홀 배판 | 켬 | **인원에 따라 자동**: 4인이면 3명 동타, 3인이면 2명 동타. 당홀에 **소급 적용** |
| 전원 동타 시 다음홀 배판 | 켬 | 4명(3인 플레이면 3명) 동타 |

| 배판 시 버디값·니어값도 배수 적용 | 켬 | 끄면 타당 정산만 배수 |
| 양파자 배판 면제 | 끔 | 켜면 배판 홀에서 양파 친 사람이 낀 쌍은 민판(×1) 정산 |

### 2.5 기타

| 항목 | 기본값 | 비고 |
|---|---|---|
| 1인 최대 손실 상한(하우스 리밋) | 없음 | 설정 시 누적 손실이 상한 도달하면 경고 표시 (정산 계산은 계속) |
| 뒷정산 메모 | — | 캐디피/밥값 등 자유 메모 (v1은 메모 수준, 자동 배분 없음) |

### 2.6 홀별 내기 스킵 (라운드 중 토글)

- 초반 몸풀기 홀 등에서 **스코어는 기록하되 해당 홀을 정산에서 제외**하는 홀 단위 토글.
- 스킵 홀은 pairwise 정산·버디값·니어·롱기·배판 트리거에서 **모두 제외**된다.
- 배판 상태도 스킵 홀을 넘어 이어지지 않는다 — **스킵 홀에서 배판 리셋** (전홀 버디로 걸린 배판이 스킵 홀을 만나면 소멸).
- 홀 입력 화면에 "이 홀 내기 제외" 스위치, 정산표에는 "제외" 표시.

## 3. 용어

| 용어 | 의미 |
|---|---|
| 타당 | 1타 차이당 주고받는 금액 (오장 = 5,000원) |
| 민판 | 배수 ×1인 일반 홀 |
| 배판 | 그 홀의 정산 배수가 ×2 (배배판 = ×4) |
| 양파 | 더블파 (파의 2배 타수) |
| 양파직전 | 더블파 - 1타 (파3 더블보기, 파4 트리플, 파5 쿼드) |
| 니어 | 파3에서 티샷을 핀에 가장 붙인 사람 |
| 롱기 | 파5에서 페어웨이에 가장 멀리 보낸 사람 |

## 4. 정산 로직 명세

### 4.1 홀별 정산용 타수 (effective strokes)

플레이어 p의 홀 h 정산용 타수:

```
eff(p, h) = min(gross(p, h), par × 2)   ← 양파 이상은 항상 양파로 (입력도 양파까지 제한)
          → 니어 "1타 차감" 모드이고 p가 유효한 니어 수령자이면 -1
```

정산용 타수는 **정산에만** 쓰이고 스코어카드(gross)와는 별개다. 판정 기준:

- 버디/이글 판정: **gross** 기준 (니어 차감으로 버디가 되지는 않음)
- 양파/양파직전 판정: gross 기준 (입력이 양파까지라 컷과 동일)
- 동타((인원−1)명/전원) 판정: **eff** 기준 (실제 정산에서 같은 돈이 오가는지의 문제이므로)

### 4.2 홀 배수 (multiplier)

배수는 홀마다 결정된다. 트리거는 두 종류:

**이월 트리거** (전 홀 → 이번 홀): 전 홀(스킵 아님)에서 발생한 다음 이벤트 각각이 +1 트리거

- 버디 이상 (설정 켬일 때, 버디한 사람 수만큼)
- 양파·양파직전 (설정 켬일 때, 해당자 수만큼)
- 전원 동타 (설정 켬일 때, +1)

**당홀 트리거** (이번 홀 자체):

- (인원−1)명 동타 (설정 켬일 때, +1 — 당홀 소급. 4인=3명 동타, 3인=2명 동타)
- 수동 배판 콜 (+1)

```
mult(h) = 중첩 곱연산 켬:  min(2^(트리거 합), 상한)
          중첩 곱연산 끔:  트리거 ≥ 1이면 min(2, 상한), 아니면 1
스킵 홀: mult = 0 취급(정산 없음), 이월 트리거 소멸, 새 트리거 미발생
```

### 4.3 쌍별 타수차 정산

홀 h에서 모든 플레이어 쌍 (a, b)에 대해:

```
pairMult = mult(h)
  단, 양파자 배판 면제 켬 && mult(h) > 1 && (a 또는 b가 양파) 이면 pairMult = 1
transfer = (eff(a,h) - eff(b,h)) × 타당 × pairMult
  → 양수면 a가 b에게 지급
```

### 4.4 보너스 (버디값/이글값/니어/롱기)

- 버디: gross ≤ par-1 인 플레이어가 **나머지 각자에게서** 버디값 수령. 이글(par-2)은 이글값, 알바트로스 이하(홀인원 포함)는 알바값. (중복 아님 — 가장 높은 등급 하나만)
- 니어(정액 모드): 유효 수령자가 나머지 각자에게서 니어 금액 수령
- 롱기: 수령자가 나머지 각자에게서 롱기 금액 수령
- "배판 시 버디값·니어값도 배수 적용" 켬이면 위 금액에 `mult(h)` 곱함

### 4.5 핸디

플레이어별 받는 타수 `h(p)`에 대해 모든 쌍 (a, b)에서 `h(b) − h(a) > 0`이면 a → b로 `(h(b) − h(a)) × 타당` 지급. 배판과 무관하며, **라운드 시작 시점부터 누적 손익에 포함**되고 정산표에 "핸디" 항목으로 별도 표기.

### 4.6 워크드 예제

설정: 타당 5,000 / 버디값 5,000 / 배판: 버디→다음홀, 중첩 곱연산 켬, 상한 ×4 / 4인 (A, B, C, D)

**3번 홀 (파4, 민판)** — A: 파(4), B: 보기(5), C: 버디(3), D: 더블(6)

쌍별 타수차 정산:

| 쌍 | 차이 | 송금 |
|---|---|---|
| A↔B | 1타 | B → A 5,000 |
| A↔C | 1타 | A → C 5,000 |
| A↔D | 2타 | D → A 10,000 |
| B↔C | 2타 | B → C 10,000 |
| B↔D | 1타 | D → B 5,000 |
| C↔D | 3타 | D → C 15,000 |

버디값: A, B, D가 C에게 각 5,000 → C +15,000

홀 합계 (타수차 + 버디값): **A +5,000 / B −15,000 / C +45,000 / D −35,000** (합 0)

**4번 홀** — 3번 홀에 C 버디 → 트리거 1개 → **배판(×2)**. 위와 같은 스코어가 나오면 모든 송금액이 2배.

**만약 3번 홀에 버디 2개**였다면 트리거 2개 → 2² = ×4 (상한 ×4에 걸려 ×4).

### 4.7 최종 정산

- 플레이어별 누적 순손익 (홀별 합 + 보너스 + 핸디)
- 쌍별 누적 상계 후, **최소 송금 요약** (예: "D → C 35,000, B → A 10,000" 식으로 송금 횟수 최소화)
- 카톡 공유용 텍스트 생성 (홀별 스코어 요약 + 최종 송금)

### 4.8 불변식

- 모든 홀·보너스·핸디 정산의 전체 합은 항상 0이다.
- 과거 홀 스코어를 수정하면 그 홀 이후의 배판 상태 포함 **전체를 처음부터 재계산**한다 (증분 갱신 없음).

## 5. 화면 플로우

```
[설정 화면] → [홀 입력 화면] ⇄ [정산표 화면] → 라운드 종료 → [지난 라운드(히스토리)]
     └──────────── 지난 라운드 보기 ────────────────────────────────┘
```

### 5.1 설정 화면
- 라운드 정보(선택): 골프장 이름, 시작 코스 — 히스토리 카드와 공유 텍스트에 표시
- **로컬룰 체크리스트 (확인용, 정산 무관)**: 도로 구제(인드롭/공 위치/골프장 룰), 티샷 OB(로컬티/두 번째로 멀리 친 사람 옆/나간 선상), 디봇 빼기, 벙커 발자국 플레이스, 언플레이어블 간소화 프리셋 제공.
  항목 텍스트 수정/삭제 가능, 직접 추가한 룰은 localStorage에 남아 다음 라운드 체크리스트에도 표시.
  체크한 룰만 라운드에 저장 → 라운드 중 상단 ⓘ 버튼으로 열람, 히스토리 상세에도 표시
- §2 체크리스트 폼. 섹션 접기/펼치기. "라운드 시작" 버튼.
- 진행 중 라운드가 localStorage에 있으면 "이어하기" 제안.

### 5.2 홀 입력 화면
- 상단: 홀 번호 네비게이션(1~18), 현재 홀 강조, **배수 뱃지(민판/배판×2/×4)** 표시
- 홀 파 선택 (3/4/5)
- 플레이어별 타수 스테퍼: 파 기준 ± 버튼 (기본값 = 파), 44px+ 터치 타깃
- 파3이면 니어 수령자 선택, 파5+롱기 켬이면 롱기 수령자 선택
- "이 홀 내기 제외" 토글, "묻고 더블" 토글(설정 켬일 때)
- 스코어 입력 완료 시 **이 홀 손익 + 여태까지의 누적 손익(핸디 포함)** 카드 표시
- 하단: 현재 누적 순손익 미니 요약 (이름: ±금액)
- 과거 홀 탭하면 수정 가능

### 5.3 정산표 화면
- 홀별 배수/스킵 표시가 있는 스코어카드 + 홀별 손익
- 플레이어별 누적 순손익, 하우스 리밋 경고
- 최종: 최소 송금 요약 + "공유 텍스트 복사" 버튼, 뒷정산 메모
- "라운드 종료 · 지난 라운드에 저장" / "저장 없이 새 라운드" 버튼

### 5.4 지난 라운드 (히스토리)
- 종료한 라운드를 기기 localStorage에 보관 (config/홀 기록 통째로 저장, 정산은 재계산. 최대 200개)
- **기본 표시는 스코어 중심** (돈이 바로 보이면 기분이 안 좋으므로): 카드에는 날짜·골프장/코스·참가자·**총타수**만.
  펼치면 홀별 스코어카드 + 메모. 돈(최종 손익/송금)은 **"오장 결과" 버튼을 눌러야** 표시
- 누적 전적 카드는 **기본 접힘**, 탭하면 펼침
- **수정하기**: 저장된 라운드를 다시 열어 진행 중 라운드처럼 스코어/니어/스킵 등 자유롭게 수정 →
  "수정 완료" 시 새 항목이 아니라 **원래 기록을 갱신**. 수정 취소하면 원본 유지.
  진행 중인 라운드가 있으면 확인 후 진입. 수정 중에는 홀 입력 화면에 "기록 수정 중" 뱃지 표시
- **누적 전적**: 같은 이름 = 같은 사람으로 보고 라운드 손익을 합산해 수익순 표시
- **백업/기기 이전**: 전체 기록 JSON 파일 내보내기 / 가져오기(같은 라운드 id는 중복 제외 병합).
  폰 교체·주소 변경·추후 서버 이전 대비
- 서버 없음 — 이 기기(브라우저)에만 저장된다는 안내 문구 표시

## 6. 기술 설계

- **스택**: React 19 + Vite + TypeScript, 의존성 최소화, 서버 없음
- **정산 엔진**: `src/engine/` — UI와 분리된 순수 함수 `settle(config, holes[]) → 정산 결과`. 룰 조합이 많으므로 vitest 단위 테스트 필수
- **상태**: `{config, holes[]}`만 저장(useReducer), 정산은 항상 파생 계산. localStorage 자동 저장/복구
- **PWA**: vite-plugin-pwa — 매니페스트 + 오프라인 캐시 (홈 화면 설치용)

```
src/
  engine/
    types.ts            # RuleConfig, HoleResult, Settlement 등
    settlement.ts       # 정산 엔진 (순수 함수)
    settlement.test.ts  # 단위 테스트
  components/
    SetupScreen.tsx
    HoleInput.tsx
    LedgerScreen.tsx
  store.ts              # useReducer + localStorage
  App.tsx, main.tsx, styles.css
```
